전체 흐름

페르소나 + 욕구 입력 (클로드코드에서 "카드 뉴스 만들어줘 --persona '20대 대학생' --desire 'AI를 더 잘 활용하고 싶다.'") │ ▼ thumbnail-writer 5개 앵글별 헤드라인 생성 + 자기 검증 루프 │ [구글시트 승인 대기] ← Human-in-the-loop (--auto 로 건너뛰기) │ ▼ body-writer 본문 3장 + CTA 생성 + 톤 일관성 검증 │ [구글시트 승인 대기] │ ▼ image-renderer Gemini 배경이미지 + 카드 5장 PNG 렌더링 │ ▼ output/{날짜}_{페르소나}/ ├── 01_thumbnail.png ├── 03_body2.png └── 05_cta.png ├── 02_body1.png └── 04_body3.png

기존과의 차이점

  • 기존 Python 기반 에이전트는 검증 로직이 에이전트마다 코드로 중복 선언
  • 하네스 방식은 evaluate-copy라는 SKILL을 정의해 썸네일 작성자와 본문 작성자가 동일한 스킬을 공유하여 검증

Agent Engineering 5가지 설계 포인트

설계 포인트 이번 사례 구현
1) 작업 단위 페르소나+욕구 1쌍 = 1 세트 (썸네일 · 본문 3 · CTA)
산출물과 로그는 output/{날짜}_{페르소나}/ 폴더에 함께 묶어 관리
2) 실행 권한 에이전트가 로컬에서 pip install · Playwright · 파일 I/O 까지 자동 수행
외부 시스템(구글 시트)은 서비스 계정으로 제한된 권한만 허용
3) 분업 각 작업을 가장 잘하는 도구에 위임
· 텍스트 카피 생성 → Claude (창의 작업)
· 글자수 · 금칙어 규칙 검증 → Python (결정적 작업)
· 배경이미지 생성 → Gemini Imagen (이미지 특화 모델)
4) 실패 복구 자기 평가 7점 미만 → 피드백과 함께 재생성 (최대 3회)
3회 모두 미달이면 최고 점수 버전 채택 → 구글 시트에서 Human이 최종 승인
5) 측정 · 비용 단계별 자기 평가 점수 · 재시도 횟수를 로그로 기록
세트당 API 비용 약 $0.025로 예측 가능 · 이상 시 상한 차단
