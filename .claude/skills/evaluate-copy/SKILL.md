# Evaluate Copy — 자기 검증 루프

카피 생성 후 품질을 보장하는 2단계 검증 스킬.

## 검증 흐름

```
카피 생성 → 1단계: 규칙 검증 → 2단계: 자기 평가 → 7점 이상? → 통과
   ↑           │ 실패            │ 7점 미만
   └────── 피드백 + 재생성 (최대 3회) ──┘
```

## 1단계: 규칙 검증

코드 없이 직접 검증한다. 아래 규칙을 카피에 대입하여 위반 항목을 찾는다.

### 썸네일 규칙
- 헤드라인이 정확히 3줄인가
- 각 줄이 한국어 기준 7~12자(공백 포함)인가
- 칩태그 2개(#으로 시작)가 존재하는가

### 본문 규칙
- 섹션타이틀이 20자 이내인가
- 인풋텍스트가 150자 이내인가
- CTA 유도문구가 존재하는가

위반 시 구체적 피드백을 생성한다 (예: "2번째 줄이 14자입니다. 12자 이하로 줄이세요."). 이 피드백을 재생성 프롬프트에 포함하여 같은 실수를 반복하지 않도록 한다.

## 2단계: 자기 평가

1단계를 통과한 카피에 대해 스스로 10점 만점으로 평가한다.

### 썸네일 평가 기준
| 기준 | 배점 | 설명 |
|------|------|------|
| 후킹력 | 2점 | 스크롤을 멈추게 하는 힘 |
| 공감도 | 2점 | 타겟이 "나 얘기다" 느끼는가 |
| 명확성 | 2점 | 3줄만으로 메시지가 전달되는가 |
| 규칙 준수 | 2점 | 글자수/줄수 형식 규칙 |
| 칩 적합성 | 2점 | 칩태그가 헤드라인/페르소나에 어울리는가 |

### 본문 평가 기준
| 기준 | 배점 | 설명 |
|------|------|------|
| 톤 일관성 | 3점 | 썸네일 헤드라인 톤과 어울리는가 |
| 정보 밀도 | 3점 | 짧은 분량에 유용한 정보가 담겼는가 |
| CTA 유도력 | 2점 | 마지막 카드가 행동을 유도하는가 |
| 규칙 준수 | 2점 | 글자수 규칙 준수 |

### 재작성

- 7점 이상: 통과
- 7점 미만: 이전 평가의 구체적 피드백을 포함하여 재생성. "이전 평가에서 후킹력 1점으로 낮았습니다. 더 강렬한 표현을 사용하세요."
- 최대 3회. 3회 모두 미달 시 가장 높은 점수 버전을 채택하고 점수를 함께 기록한다.
